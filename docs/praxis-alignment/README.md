# Praxis File Management x Praxis Platform

Este diretório centraliza o material de alinhamento arquitetural entre `praxis-file-management` e a plataforma Praxis.

Objetivo:
- registrar o diagnóstico atual;
- manter um plano de execução por fases;
- manter um backlog operacional por repositório;
- preservar o contexto desta conversa para retomada após interrupções.

Arquivos ativos:
- `STATUS.md`: estado atual, próximos passos e checkpoints de retomada.
- `EXECUTION_PLAN.md`: plano concreto por fases.
- `OPERATIONAL_BACKLOG.md`: épicos, histórias e tarefas técnicas por repositório.
- `COMPATIBILITY_MATRIX.md`: estado atual de compatibilidade entre backend e frontend após o corte do contrato backend.
- `ENVELOPE_ALIGNMENT.md`: shape final atual do web layer e diferenças remanescentes para aproximar o módulo da Praxis.
- `CONVERSATION_LOG.md`: resumo estruturado da investigação e das decisões tomadas.

Arquivos históricos:
- `history/HISTORICAL_TRANSITION_CONSOLIDATION_CHECKLIST.md`: checklist da fase transitória, hoje superada.
- `history/HISTORICAL_FUTURE_MIGRATION_PROPOSAL.md`: proposta ligada à estratégia transitória anterior.
- `history/HISTORICAL_PRAXIS_CONVERGENCE_NEXT_MILESTONE.md`: plano curto anterior à decisão de remover legado.

Leitura recomendada:
1. `STATUS.md`
2. `EXECUTION_PLAN.md`
3. `COMPATIBILITY_MATRIX.md`
4. `ENVELOPE_ALIGNMENT.md`
5. `OPERATIONAL_BACKLOG.md`
6. `CONVERSATION_LOG.md`

Observação:
- os arquivos históricos permanecem apenas para contexto em `history/`. Eles não devem ser usados como fonte primária de retomada, porque refletem uma estratégia transitória abandonada quando foi decidido remover legado antes da entrada em produção.
